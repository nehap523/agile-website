Critical reflection on how AI was used during this project

This page documents how AI tools were used during development, the prompts I used, what the AI produced, what I changed, and what limitations, reliability concerns, risks, and ethical issues became clear during the process.

Use Case 1

Using AI for initial planning and idea validation

Example prompt

“I have a four-page web project with a tutorial page, quiz page, AI reflection page, and CV page. Suggest a page structure that fits the assignment brief.”

How it helped

The AI suggested a four-page structure, a consistent navigation flow, and a modular file organisation using separate HTML, CSS, JavaScript, and data files. This helped me translate the assignment brief into a practical implementation plan more quickly.

My contribution

I compared those suggestions with the assignment brief and rubric, simplified parts that were too broad, and decided which ideas stayed within the lecture scope. The final structure, design decisions, and implementation details were all chosen and written by me.

Critical evaluation

AI was helpful for planning, but its suggestions were high-level and sometimes too confident. It could propose a good-looking structure without guaranteeing that every feature matched the rubric exactly. This meant I still had to interpret the requirements myself and decide what was realistic, relevant, and technically appropriate.

Use Case 2

Using AI for debugging and clarification

Example prompt

“My quiz page is loading but the questions are not appearing correctly from the JSON file, and some navigation or file path issues are breaking the page. What should I check first?”

How it helped

AI suggested likely causes such as incorrect relative file paths, JSON loading problems, and JavaScript logic mistakes. It also broke the debugging process into smaller steps, which made the issues easier to investigate.

My contribution

I tested each suggested fix in the browser, checked the console, corrected the file structure myself, and confirmed that the final solution still matched lecture topics such as DOM manipulation, event handling, AJAX, and local storage.

Critical evaluation

This use of AI highlighted a reliability issue: AI could produce plausible debugging advice that was not always the real cause of the problem. I still had to verify each suggestion by testing it, because a confident answer from AI did not guarantee correctness. In that sense, AI supported debugging, but it did not replace the need for understanding the code myself.

Use Case 3

Using AI for validation and concept understanding

Example prompt

“Compare my current implementation with the assignment requirements and rubric. Tell me what is still missing, especially for AJAX, local storage, quiz validation, and the AI reflection criteria.”

How it helped

AI helped identify possible gaps between my implementation and the rubric, especially around quiz validation, reward fetching, local storage, and reflection requirements. It also explained some concepts in simpler language when I needed a quicker clarification before continuing with implementation.

My contribution

I reviewed those suggestions critically, compared them with the actual rubric, and then decided which changes were necessary. I only applied changes after I understood why they were needed and how they worked in my own code.

Critical evaluation

AI was useful for validation, but it could still miss important details or overstate how complete a feature was. This meant I could not rely on it as a final judge of correctness. I still needed to test features, read the brief carefully, and confirm that the implementation actually worked as required.

Critical Review

One case where AI output was incorrect and how I detected it

Specific example

In one case, AI suggested a page structure and styling approach that initially placed too much CSS directly inside the HTML file instead of keeping the project cleanly modular with separate CSS and JavaScript files.

What was wrong

The suggestion looked functional at first, but it was not fully aligned with the assignment expectation that the final submission should contain clearly separated HTML, CSS, and JavaScript files. It also made the page harder to maintain and less consistent with good web development practice.

How I detected it

I detected the issue by comparing the generated structure with the assignment brief, checking the project folder organisation, and reviewing whether the solution reflected the separation of concerns taught in lectures. I then moved the page-specific styles and logic into dedicated CSS and JavaScript files.

Why this mattered

This example showed that AI output can appear correct because it works visually, even when it is not the best solution academically or structurally. That made it important to judge AI output against both technical correctness and assignment expectations.

Evaluation

Reliability, risks, and ethical considerations

Reliability

AI tools were useful for producing quick suggestions, but they were not fully reliable. Some responses were accurate, while others were incomplete, too general, or overly confident. I therefore treated AI output as a starting point rather than a final answer.

Risks

One major risk is over-reliance. If I had copied AI output without understanding it, I would have struggled to debug it, explain it in the demo, or adapt it to the rubric. Another risk is that AI can encourage false confidence by making an incomplete solution sound correct.

Ethical considerations

Ethical use of AI in a coding project requires transparency, academic honesty, and independent understanding. AI can support learning, but it should not replace the student’s own thinking or be presented as original work without acknowledgement. In this project, I used AI as a support tool and ensured that final decisions, coding, corrections, and testing remained my responsibility.
