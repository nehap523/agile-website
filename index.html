Learn HTML, CSS, and JavaScript in one clear, interactive guide.

Web pages are built by combining three core technologies. HTML gives the page structure, CSS controls how it looks, and JavaScript adds behaviour and interactivity. This tutorial introduces the fundamentals, shows practical code examples, and includes mini demonstrations so you can see each concept in action.

HTML builds the meaning

HTML describes the structure of a page using headings, paragraphs, lists, images, forms, and semantic layout elements such as header, nav, section, article, and footer.

CSS shapes the presentation

CSS controls colour, spacing, layout, responsiveness, and visual hierarchy. It helps keep content separate from presentation, which improves maintainability and consistency.

JavaScript adds behaviour

JavaScript responds to events, updates the DOM, validates input, stores local data, and communicates with files or APIs through asynchronous requests.

How the three technologies work together

Keeping structure in HTML, presentation in CSS, and behaviour in JavaScript follows the separation of concerns approach and makes the site easier to read, debug, and extend.

HTML fundamentals

HTML describes the content and structure of a web page.

HTML uses elements such as headings, paragraphs, links, lists, images, tables, and forms to organise information. Modern HTML also includes semantic elements like <header>, <nav>, <main>, <section>, and <footer>, which make the meaning of the page clearer for developers, browsers, and assistive technologies.

  • Use headings to show content hierarchy.
  • Use semantic tags to improve readability and accessibility.
  • Use forms to collect input from users in a structured way.
Why best practice matters: semantic HTML is not just about passing validation. It also makes content easier to navigate, improves accessibility, and gives your CSS and JavaScript cleaner hooks to work with.
<header> <h1>My Portfolio</h1> <nav> <a href="#projects">Projects</a> <a href="#contact">Contact</a> </nav> </header> <main> <section id="projects"> <h2>Featured Projects</h2> <p>Here are some examples of my work.</p> </section> </main>

CSS fundamentals

CSS controls style, spacing, layout, and responsive presentation.

CSS works by applying rules to selected HTML elements. A selector targets an element, then property-value pairs define its appearance. Using external CSS keeps design rules in one place, which makes updates faster and helps maintain a consistent visual identity.

  • Selectors decide which elements are styled.
  • The box model controls margin, border, padding, and content size.
  • Responsive design helps a page adapt across phones, tablets, and desktops.
Why best practice matters: relying only on framework defaults can make your site look generic. Custom CSS helps create a distinctive feel and demonstrates stronger design skill.
body { font-family: Arial, sans-serif; background-color: #f8fafc; color: #1f2937; } .card { border: 1px solid #dbe4ee; border-radius: 16px; padding: 24px; box-shadow: 0 12px 30px rgba(0, 0, 0, 0.08); } .card:hover { transform: translateY(-4px); }

JavaScript fundamentals

JavaScript makes a page interactive by responding to events and updating the DOM.

JavaScript can react to button clicks, form input, mouse movement, and page load. It can also select elements from the document and change text, classes, styles, or structure at runtime. This is what turns a static page into an interactive application.

  • Events include click, input, submit, mouseover, and load.
  • The DOM is the browser's object model of the current HTML page.
  • JavaScript can also fetch data asynchronously from JSON files or APIs.
Why best practice matters: Keep JavaScript organised into readable functions. Clean logic is easier to debug, easier to explain in a demo, and easier to extend later.
